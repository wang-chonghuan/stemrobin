# STEMROBIN-13 — 统一数学课练习为选择题

## Meta
- Type: story
- Batch: 0003-math-choice-exercises
- Origin: seed
- Seed: STEMROBIN-12

## Scope
学习者打开任一现有或后续生成的数学课时，课文内“练习”和“卡片答题”中的每一道练习题都可通过选择选项作答；一题可以提供超过四个选项。

## Constraints
- 仅改数学课程练习；传记章节题不在本工单范围内。
- 当前 11 节数学课的题干文字和题序必须保持不变；每题只补充可选项并保留唯一正确项。
- 新生成的数学课程练习必须全部为选择题；课文、打印版和卡片答题必须呈现同一组题目与选项。
- 在替换现有数学题库前清除现有数学答题记录和答题尝试，不迁移历史数据（人类裁决 2026-07-12）。
- 答案、正确选项与参考讲解在作答前不得暴露给客户端。

## Acceptance criteria
- 打开每一节当前数学课的课文和“卡片答题”，每一道练习均以可点击选项作答；不再出现填空输入或自我核对的问答作答方式。
- 当前数学课中每一道练习的题干文字和题序与改造前一致，同时每题都有选项且恰有一个正确答案。
- 通过更新后的生成流程保存一节数学课时，生成的每道练习均为选择题；包含五个或更多选项的题目会完整出现在课文、打印版与卡片答题中，并可被正确判分。
- 改造完成后，旧的数学答题记录和答题尝试已被清除；新完成一次数学课答题后会正常记录得分，且作答前不会显示答案。

## Product charter

### Goal

为一名 8 岁、理解力强的孩子提供初中数学/物理的自学课程产品。内容按初中标准，解释按儿童认知，训练按严肃教材；练习题系统包含识别、表示、基础操作、反向推理和易错辨析。

### Engineering rules

- 只做完成本工单所必需的改动，保留单一事实来源，并以实际运行验证结果。
- 内容由项目技能生成并通过相应 saver 持久化；不得手写 `sr_*` 数据库行。
- 所有数据库读写都通过服务器端 `sql()`；不得泄露连接信息。
- 初始题目读取不得把 `correct_index`、`answer` 或 `accept` 发到客户端；服务端在作答后判定正确性。
- 不添加依赖；使用当前架构与工具。

### Architecture and runbook

- 数学课由 `sr-math-lesson` 生成，题组保存到 `sr_questions`；保存器从题组生成课文练习区和打印 PDF。
- 应用为 `app/` 下的 TanStack Start 项目；开发、测试和构建命令在 `app/` 目录运行。
- 内容技能从 `.agents/skills/` 运行，通过其 saver 写入数据库。数据库 schema 的唯一来源是 `ssot-schemas/db-schemas/stemrobin.sql`。
- 运行 `cd app && npm run test`、`npm run build` 和 `npm run e2e`；本地开发为 `cd app && npm run dev`。
- 根 `.env` 是本地数据库与运行时配置，不能提交；`app/.env` 应链接到根 `.env`。

### Redlines

清除现有数学答题记录和答题尝试已获得人类明确批准；不得执行其他未经批准的破坏性或外部系统操作。

## Relevant current-state facts

- 当前有 11 节数学课和 233 道题：59 道 `choice`、143 道 `input`、31 道 `work`。
- 数学题组替换会删除旧题并级联删除相连答题事件；用户已批准清除历史记录。
- `sr-math-lesson` 的题组契约和校验器目前要求混合 `choice`、`input`、`work`；保存器只把 choice 的选项投影到课文练习区。
- 卡片答题 UI 已可渲染任意数量的 choice 选项，并由服务端按隐藏 `correct_index` 判定。
- 传记章节题使用独立题表和生成技能，不在本工单范围内。
