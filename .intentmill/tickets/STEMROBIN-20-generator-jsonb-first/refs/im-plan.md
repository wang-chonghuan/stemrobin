## Implementation Drift Controls

- **Answer-key secrecy cannot be bypassed.** The KEY (`correct_index`/`accept`/`answer`) is written ONLY into neutral-base `key`. `check-content.mjs` must fail if the overlay contains any KEY field; the render projection must never serialize `key`. cap6 must grep the rendered HTML and prove zero KEY leakage (mandatory, not best-effort).
- **Ledger must come from the DB.** The content/exercise authoring + validators read the ledger from `sr_content_ledger`; completing the sample run must not open any `resources/content/math-ledger/*.json`. cap6 verifies this (e.g. the run works with the local ledger files absent/irrelevant, and no code path reads them during authoring).
- **JSONB shapes are the documented contract.** `content`/`exercises`/`read_check`/`key`/`rev`/overlay must match `ssot-schemas/db-schemas/stemrobin.sql` exactly; Postgres does not enforce internals, so `check-content.mjs`/`check-exercises.mjs` are the enforcement point and must run before every DB write.
- **Genre anchors preserved.** The card-tree anchors and their order must equal the existing per-genre set; do not invent or reorder anchors.
- **Closure + composition preserved.** Do not weaken the prerequisite-closure or deck-composition rules; they run against the DB-sourced ledger.
- **Derived cache, no app change.** The saver must keep populating a valid `sr_lessons.html`; do not modify `app/` or `curriculum.ts`. If the reader would break, fail fast and record it ÔÇö do not silently change app code.
- **Rejected option stays absent.** Do NOT inline learner-visible prose in `content.body`; prose lives in the `zh` overlay (D-OVERLAY).
- **Prod-data safety.** All sample writes use a disposable id/stage; sample rows are deleted after proof; `sr_users` and the 16 real lessons are never touched. Any DB write is scoped to the sample id/stage only.
- **External-dependency fail-fast.** If the shared Azure Postgres is unreachable / creds invalid, or if playwright-core cannot launch a browser engine (so a real PDF cannot be produced for acceptance), stop and record a `## Blocker` in `im-handoff.md`; do not mock or fabricate a PDF.
