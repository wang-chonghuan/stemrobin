# im-spec — STEMROBIN-36 · login-logout

## Goal
A clean **login page** with no protected content, and a **visible logout** control, reusing the
existing HMAC session. No registration. `app/` only, no new dependency, `sr_users` untouched.

## Behaviour (black-box)
1. **Bare login page.** A logged-out visitor at `/login` sees only a login card (brand + email +
   password + submit + error slot). No catalog sidebar, no lesson/curriculum titles, no other
   protected content.
2. **Login works.** Submitting valid credentials calls the existing `login` server fn and, on
   success, enters the app (`/`). Invalid credentials show the existing server error message.
3. **Login gate stays.** A logged-out visitor to any protected surface (`/`, `/lesson/$id`) is
   redirected to `/login` (STEMROBIN-31 behaviour, unchanged in effect).
4. **Visible logout.** Inside the app, the catalog sidebar footer shows the signed-in email and a
   logout button. Clicking it calls the existing `logout` server fn (clears `sr_session`) and
   returns to `/login`. Afterwards, protected pages require login again.
5. **No registration.** No signup / create-account entry on the login page or anywhere in the app.

## Design / structure
- **Login route** moves from `_app/login.tsx` (gated shell child) to top-level
  `app/src/routes/login.tsx` (child of `__root`, sibling of `_app`) → renders with no `_app`
  shell. Self-contained centered card. Same form logic (email/password state, busy guard, calls
  `login`, `router.navigate({to:'/'})` on success). If already logged in, redirect to `/` in
  `beforeLoad` (so the bare login page is not shown to an authenticated user).
- **`_app.tsx` gate** simplifies: drop the `/login` pathname exception; the gate now uniformly
  requires a user for every `_app` surface. Loader additionally fetches `getCurrentUser()` and
  passes the user to `AppShell` → `CatalogSidebar`.
- **`CatalogSidebar`** gains a footer (`.sr-cat-foot`) rendering the user email + a logout button
  (`.sr-logout`). Handler: `await logout(); router.navigate({ to: '/login' })`.
- **CSS** (`app/src/styles/app.css`): add `.sr-auth` centered full-viewport wrapper for the bare
  login page + a `.sr-auth-card`; add `.sr-cat-foot` / `.sr-logout`. Reuse existing `.sr-login*`
  form styles and `--sr-*` tokens only (no new hues — DESIGN.md three-color palette).
- **i18n** (`app/src/lib/i18n.ts`): add `login.logout` (zh 登出 / en Sign out). Optional
  `login.signedInAs` label for the email row.

## Constraints / invariants
- Reuse `login` / `logout` / `getCurrentUser` from `app/src/lib/session.ts`; do not add a session
  fn or touch session crypto.
- Never touch `sr_users`. No new dependency. `app/` only. Do not hand-edit `routeTree.gen.ts`
  (regenerated by the Start plugin on dev/build).
- Keep the STEMROBIN-31 gate effect: unauthenticated protected access → `/login`.

## Acceptance mapping
- AC "登录页登录后进入应用" → Behaviour 2.
- AC "应用内有可见登出操作，点击后退出并回到登录页" → Behaviour 4.
- AC "无注册入口" → Behaviour 5.
- AC "登出后访问受保护页被要求重新登录" → Behaviour 3 + 4 (gate re-applies after cookie cleared).
- Constraint "登录页是干净登录界面、不暴露受保护内容" → Behaviour 1.

## Verification (gate6, empirical browser)
Standalone Playwright (`app/node_modules/playwright`) against `npm run dev`:
- (a) No cookie → GET `/login`: assert login form present AND no `.sr-catalog` / no lesson titles.
- (b) Mint user-2 `sr_session` cookie (no password typed) → GET `/`: assert app shell + visible
  logout control present.
- (c) From logged-in app, click logout → assert URL is `/login`, bare page; then GET `/` with the
  post-logout context → assert redirected back to `/login` (gate re-applied).
- (d) Assert no register/create-account text anywhere on `/login`.
Plus `cd app && npm run test` and `npm run build` clean.
