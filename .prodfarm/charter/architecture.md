# Architecture

- Single tanstack-start app; SSR routes under `src/routes/_app/` (index = catalog, `lesson.$id`, `story.$id`, `login`).
- Domain libs in `src/lib/`: `curriculum.ts` (course structure + lesson ordering/nav), `lessons.ts`, `stories.ts`, `quiz.ts` + `answer-normalize.ts` (practice, incl. `input`-mode server-side judging), `session*.ts` (auth), `db.ts` (postgres access, `search_path` = the project schema).

## Stack & constraints

- **Base**: tanstack-start (SSR full-stack, single app in `src/`), React 19, TypeScript. This is the one supported n-easyapp base for this repo; a second app / a framework change is out of scope without a charter decision.
- **Routing**: `@tanstack/react-router` (file-based, `src/routes/`).
- **Styling**: Tailwind CSS 4 (+ tw-animate-css), Geist / Bricolage / Hanken / JetBrains Mono fonts. Three-color palette per `DESIGN.md` (teal-blue, green, white) â€” no new hues.
- **State**: zustand (`src/lib/layout-store.ts`).
- **Data**: PostgreSQL via the `postgres` client (`src/lib/db.ts`); the DB is the Azure easy-app **shared** server, per-project schema `stemrobin-schema` â€” never a second client, never scale-to-zero. Session via `src/lib/session.server.ts` (HMAC cookie).
- **Content pipeline**: markdown via `marked`; lessons/stories/quiz served from the DB through `src/lib/`.
- **Build/test**: vite, vitest (`npm run dev|build|test|start`).
- **Deploy**: Azure Container Apps `ca-stemrobin` via n-easyapp; **Dockerfile at repo root, build context = repo root** (n-easyapp's redeploy hard-codes both). The in-repo content skills (`sr-math-lesson`, `sr-story`) resolve their npm deps from **repo-root `node_modules`** â€” so the app manifest + `node_modules` live at repo root (moving them requires an npm workspace to keep root resolution working). Recorded so a future layout change does not silently break deploy or the content scripts.
