# Batch 0009 grill（cap11 自裁 + 人已同意的四点）

seed 人类发起、full delegation。核心是把课程生成从“文本+KaTeX 为主（代数思维）”升级为“**按领域自适应、图作为一等表示**”。四点 grill 已与人拧清并同意：

1. **不预设=必要性检验，不是计数**：agent 在蓝图显式写图判断+理由；检验按“依赖图才成立的论断/题都配图、且无废图”卡，不用 `≥N`。（否则“自由裁量”会退回每课一张图的老样子。）
2. **练习题带图必跨 skill+app+schema**：app 现在把题干当纯文本转义、sr_questions 无图列——不是纯 skill 能收口。
3. **反向验证**：新流水线拿 3.5 代数课 dry-run（不保存），证明它按领域自适应、不误加图。
4. **图工具锁 build 期 spec→SVG**（非运行时交互库）：静态、PDF 安全、坐标由工具算（根治手写坐标画错图）。不做交互图（另一个更大方向）。

cap11 自裁：四点均立于实测（当前渲染/schema/app 的转义与缺字段已核实）；无待人裁的遗留产品决策（交互图已明确排除）；无 redline（改 skill/app 代码 + schema 加列 + 重生成内容；老课数据不动）。
DAG 与依赖清晰，风险集中在 T1 图工具的坐标正确性与 T3 的 app 渲染注入——都以实测验收把关。

裁决：4 单（3 enabler + 1 chore）成立，full delegation 开发。
