## cap11 自评

- **可行性**：全部为本地脚本与技能内数据，无新外部依赖、无新计费资源。SQLite 用 node 内置能力或既有依赖即可，不引入新库则更佳（宪章：无谓依赖为铁律）。
- **一致性**：与宪章 SSOT 的张力已在会话中与人辩明并达成一致——**覆盖率等可推导数字仍现算，状态库只存推导不出来的决策与过程**（改嫁去向与理由、生成历史、处置状态）。人另裁定 SQLite 入 git，代价是二进制不可 diff，故要求同时导出可审快照。
- **完整性**：本批解决的是「计划与实际脱节」这一结构性缺陷，证据为实测（2 课漏 16 词，外推 70 课约 480 词）。若不先修，量产必然产出静默残缺的课程。
- **批次风险**：95 与 99 存在互为前置的表象；已在 story-list 中以两阶段（初稿难词 → 蓝图定稿 → 分配定稿）解开，不构成死锁。

## 追加裁决（人类 · 2026-07-22）：词表整体更换

调查触发点：94 的体检发现 **VOA1500 是新闻词表**——收 administration/guerrilla/asylum，却不收 breakfast/park/sorry/phone。蓝图 437 个要求词中 119 个不在表内（36%），导致「第13课叫家里的早餐却不能写 breakfast、第22课在公园却不能写 park、第27课教婉拒却不能写 sorry」。机器先按人的指示补了 147 个日常词，随后人提供牛津 3000 词表要求重新评估。

实测比较（证据而非印象）：

- 抽取工具：markitdown 与 pdftotext -layout **结果完全等价**（同为 2968 词条、CEFR 零分歧）。真正的关键不是工具，而是**按 CEFR 标记切分词条**的解析策略，对两种抽取都成立。
- 自查揪出解析器三个 bug，曾静默丢失约 45 个词条（含 light/minute/can/do/like），成因是括号消歧+跨行续行、上标同形异义数字、多义词取了最高而非最低等级。修正后 2938 → 2970。
- 规模：牛津 A1+A2 = 1662 词，与「VOA1541 + 补充147 = 1688」几乎相同，但覆盖蓝图 33 个关键场景词的 **100%**（且全部为 A1），而 VOA 一个都没有。
- 机器此前手工补的 147 个词，牛津自带 124 个（84%）。

裁决：

1. **彻底废弃 VOA1500，改用牛津 3000 的 A1+A2**。丢弃的 272 个 VOA 新闻词正是不该教给 12 岁孩子日常英语的部分。CEFR 等级还顺带解决了蓝图「难度由具体到抽象」的排序，不再需要机器用关键词打分去猜。
2. **状态只用一份 JSON，不要 SQLite**。机器原以 SSOT 为由主张「库为真相 + 导出快照」，但那是针对「静态表 + 派生库」两个真相源；只有一份 JSON 时它本身即唯一真相，且可 diff 可审，对 1712 条数据完全够用——机器接受此判断并承认原方案是误判。
3. **牛津原表完整存一份于仓库**，以后不再解析 PDF。
4. 蓝图要求而牛津没有的词（autumn/cough/sidewalk/o'clock/sunny/rainy/windy/salty/sour/tidy/hug/junk/okay/english 共 14 个）**由机器补充，但必须标注来源字段**说明是机器所加及理由。
5. **UI 与代码中不得再出现 VOA1500**，另立清理工单（103）；技能改以其产出物命名，避免将来换词表又要改名。
6. 课程规模按新词表重算：约 1520 个上卡词 ÷ 每课 22 词 → **12 单元 × 7 = 84 课**。
